 UNICHAT - TEST ACCOUNTS AND HOW TO LOG IN
===============================================================================
 All accounts below are created by the seed script. They are DEV credentials
 for local testing only - do not reuse them anywhere real.

 Every account belongs to the demo institution:
     Arusha Technical College   (slug: atc)


-------------------------------------------------------------------------------
 0. BEFORE YOU CAN LOG IN
-------------------------------------------------------------------------------
 Your database is your own - it starts empty and none of these accounts exist
 until you seed it. Run these once, from the repo root:

     docker compose up -d --build            (start backend, first run is slow)

     cd infrastructure\database
     npm install
     npm run migrate                         (creates the tables)
     npm run seed                            (creates the accounts below)

 Check the backend is alive before going further:

     curl http://localhost/api/v1/auth/institutions

 That must return JSON containing "Arusha Technical College". If it fails,
 nothing else will work - see TROUBLESHOOTING at the bottom.

-------------------------------------------------------------------------------
 2. WHERE TO SIGN IN
-------------------------------------------------------------------------------
 The web admin app serves BOTH admin dashboards. They are separate logins
 with separate cookies:

     http://localhost:3000              -> SuperAdmin login (password)
     http://localhost:3000/admin-login  -> ATC admin login  (OTP)

 To start it:

     cd apps\web-admin
     npm install
     npm run dev

 The mobile app is for students:

     cd apps\mobile
     flutter run

 On first launch pick "Arusha Technical College", then enter a student email.
 If the app cannot reach the backend, check the server field at the bottom of
 the screen:
     - running the app on this same PC (Chrome/Windows) -> localhost
     - running on a phone over Wi-Fi -> this PC's LAN IP (run: ipconfig)

 NOTE: apps/web-client and apps/web-superadmin are empty placeholder scaffolds.
 There is nothing to run in them. The superadmin UI lives inside web-admin.


-------------------------------------------------------------------------------
 3. GETTING THE OTP CODE  (needed for students AND the ATC admin)
-------------------------------------------------------------------------------
 There is no email or SMS provider wired up, so login codes are PRINTED TO THE
 BACKEND LOG. Keep this open in a second terminal while logging in:

     docker logs -f unichat-auth-service

 To filter it:

     docker logs -f unichat-auth-service | findstr /I "OTP"

 You will see one of:

     [OTP] Test Student (student@atc.ac.tz): 483920          <- student login
     [ADMIN OTP] ATC System Administrator @ atc: 592014       <- admin login

 Codes expire after 5 MINUTES. Request a new one if it lapses.


-------------------------------------------------------------------------------
 4. TESTING CHAT BETWEEN THE TWO STUDENTS
-------------------------------------------------------------------------------
 You need both students signed in AT THE SAME TIME, on DIFFERENT devices -
 one session per device. For example:

     Student 1 -> your phone      (flutter run, pick the phone)
     Student 2 -> Chrome on PC    (flutter run, pick Chrome)

 Then from one of them: search for the other student, open a chat, and send.

 What should work:
     - messages appear on the other device instantly
     - "is typing..." shows while the other person types
     - photos send and appear in both chats
     - reply to a message (long-press -> Reply) survives a reload
     - turn Wi-Fi off: saved chats still open, and a message you send is
       queued with a clock icon, then sends itself when Wi-Fi returns

 LIMITATION: students CANNOT create groups - that is deliberate and returns
 403. Group and course creation is restricted to teachers/admins. To test a
 group, create it while signed in as admin@atc.ac.tz and add both students.


-------------------------------------------------------------------------------
 5. TROUBLESHOOTING
-------------------------------------------------------------------------------
 "npm : File ... npm.ps1 cannot be loaded ... running scripts is disabled"
     PowerShell blocks npm. Either use cmd.exe instead, or run:
         Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

 "Environment variable not found: DATABASE_URL"
     infrastructure/database/.env is missing. It ships with the repo - make
     sure you pulled the latest and are running the command from inside
     infrastructure\database.

 "Cannot find module '.prisma/client/default'"
     The Prisma client was not generated (npm 11 blocks install scripts).
         cd infrastructure\database
         npx prisma generate

 POST /api/auth/login returns 500 in the web admin
     auth-service is not running, so nginx returns an error page the app
     cannot read. Check it:
         docker compose ps
         docker compose logs --tail=30 auth-service
         docker compose up -d --build

 "Filename too long" while cloning or pulling
     The repo contains build output with deep paths. Enable long paths:
         git config --global core.longpaths true
     Also clone somewhere short, e.g. C:\Unichat, not a deeply nested folder.

 Mobile app says "Network error"
     - is the backend up?   curl http://localhost/api/v1/auth/institutions
     - on a phone, is the server field set to this PC's LAN IP, not localhost?
     - is the phone on the SAME Wi-Fi as the PC (not mobile data)?
     - Windows Firewall may block port 80 from other devices. As admin:
         New-NetFirewallRule -DisplayName "UniChat Gateway (TCP 80)" `
           -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow -Profile Any

 An image in chat shows a broken placeholder
     That photo's file was deleted from the server; the message still points
     at it. Send a new one - it will be fine.
